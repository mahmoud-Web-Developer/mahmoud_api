========================================
   حالة Railway API
========================================

## الـ Base URL
https://mahmoudapi-production.up.railway.app

## اختبار سريع

### 1. اختبار صحة النظام:
```bash
curl https://mahmoudapi-production.up.railway.app/health
```

### 2. اختبار معلومات الـ API:
```bash
curl https://mahmoudapi-production.up.railway.app/api
```

### 3. اختبار تسجيل الدخول:
```bash
curl -X POST https://mahmoudapi-production.up.railway.app/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}'
```

### 4. اختبار لوحة التحكم (مع Token):
```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://mahmoudapi-production.up.railway.app/dashboard
```

## الـ APIs المتاحة

### APIs عامة (لا تحتاج مصادقة):
- GET /health - صحة النظام
- GET /api - معلومات الـ API
- GET /services - الخدمات
- GET /portfolio - البورتفوليو
- GET /news - الأخبار
- POST /contact-requests - طلب تواصل
- GET /meetings/slots - أوقات الاجتماعات
- POST /meetings - حجز اجتماع
- POST /briefs - إرسال Brief
- POST /auth/signup - تسجيل مستخدم جديد
- POST /auth/login - تسجيل الدخول

### APIs محمية (تحتاج JWT Token):
- GET /dashboard - لوحة التحكم
- POST /clients - إضافة عميل
- GET /clients - جلب العملاء
- GET /clients/:id - جلب عميل محدد
- POST /requests/contact - بدء طلب تواصل
- POST /requests/meeting - بدء طلب اجتماع
- POST /requests/brief - بدء طلب Brief

## حالة النشر

### إذا كانت التغييرات لم ترفع بعد:
1. رفع التغييرات:
```bash
git add .
git commit -m "fix: إصلاح مشكلة المصادقة"
git push origin main
```

2. انتظار النشر:
Railway سيقوم بالنشر التلقائي خلال 2-5 دقائق

3. اختبار النشر:
```powershell
.\check-railway.ps1
```

## استكشاف الأخطاء

### إذا كان الـ API لا يعمل:
1. تحقق من حالة Railway في لوحة التحكم
2. تحقق من الـ logs في Railway
3. تأكد من أن التغييرات تم رفعها
4. انتظر 5 دقائق للنشر التلقائي

### إذا كانت المصادقة لا تعمل:
1. تأكد من وجود مستخدم admin
2. تأكد من صحة JWT secret
3. تأكد من إضافة middleware المصادقة
